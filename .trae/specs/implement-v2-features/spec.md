# SilkRoad-Next V2 性能优化与功能增强规范

## Why

在 V1 MVP 基础代理功能之上，V2 版本旨在通过引入连接池、线程池、会话管理、缓存管理、黑名单拦截和脚本注入等高级功能，大幅提升系统性能、安全性和可定制性，支持更高并发场景并提供更灵活的功能扩展能力。

## What Changes

- 实现目标服务器连接池管理（modules/connectionpool.py）
- 实现 CPU 密集型任务线程池（modules/threadpool.py）
- 实现客户端会话管理系统（modules/sessions.py）
- 实现双层缓存管理器（modules/cachemanager.py）
- 实现多层次黑名单拦截系统（modules/blacklist.py）
- 实现前端脚本注入系统（modules/scripts.py）
- 创建黑名单配置文件（databases/blacklist.json）
- 创建脚本注入配置文件（databases/scripts.json）
- 创建脚本库目录和示例脚本（Scripts/）
- 创建缓存存储目录（cache/）

## Impact

- **新增功能**：
  - 连接池：降低 TLS 握手开销，提升连接复用率
  - 线程池：防止 CPU 密集型任务阻塞事件循环
  - 会话管理：支持客户端状态持久化
  - 缓存管理：双层缓存架构，LRU 淘汰策略
  - 黑名单拦截：IP/URL/域名多层次访问控制
  - 脚本注入：支持前端功能增强和定制化

- **性能提升**：
  - 连接复用率提升 60%+
  - 缓存命中率 70%+
  - CPU 密集型任务处理时间减少 40%+
  - 支持 5000+ 并发连接（V1 为 2000+）

- **影响范围**：
  - 扩展现有模块功能
  - 新增 6 个核心模块
  - 新增配置文件和数据文件
  - 无破坏性变更（向后兼容 V1）

## ADDED Requirements

### Requirement: 连接池管理系统

系统 SHALL 提供目标服务器连接池管理功能：
- 支持按主机名维护连接池
- 支持最大连接数限制（每主机默认 10 个）
- 支持连接超时检测（默认 60 秒）
- 支持连接健康检查
- 支持连接复用和归还
- 支持自动清理过期连接
- 支持连接统计信息查询

#### Scenario: 连接复用成功
- **WHEN** 客户端请求目标网站
- **THEN** 系统从连接池获取可用连接
- **AND** 连接被正确复用
- **AND** 连接使用计数增加

#### Scenario: 连接池已满
- **WHEN** 连接池达到最大连接数
- **THEN** 系统等待连接释放或创建新连接
- **AND** 记录连接池状态日志

#### Scenario: 连接超时清理
- **WHEN** 连接超过 Keep-Alive 超时时间
- **THEN** 连接被自动关闭
- **AND** 从连接池中移除
- **AND** 清理计数增加

### Requirement: 线程池管理系统

系统 SHALL 提供 CPU 密集型任务线程池：
- 支持可配置的工作线程数（默认 CPU 核心数 * 2）
- 支持任务超时控制
- 支持任务优先级调度
- 支持批量任务执行
- 支持任务统计信息查询
- 支持优雅关闭

#### Scenario: CPU 密集型任务执行
- **WHEN** 需要执行大规模正则替换或解压缩
- **THEN** 任务被提交到线程池
- **AND** 不阻塞主事件循环
- **AND** 任务完成后返回结果

#### Scenario: 任务超时处理
- **WHEN** 任务执行超过超时时间
- **THEN** 任务被取消
- **AND** 返回超时错误
- **AND** 记录超时日志

#### Scenario: 批量任务执行
- **WHEN** 提交多个独立任务
- **THEN** 任务并发执行
- **AND** 结果按顺序返回
- **AND** 统计总执行时间

### Requirement: 会话管理系统

系统 SHALL 提供客户端会话管理功能：
- 支持会话创建和销毁
- 支持会话数据存储（最大 1MB）
- 支持会话超时自动清理（默认 30 分钟）
- 支持会话持久化到文件
- 支持按 IP 地址查找会话
- 支持会话统计信息查询

#### Scenario: 会话创建
- **WHEN** 新客户端首次访问
- **THEN** 创建唯一会话 ID
- **AND** 记录客户端 IP 和 User-Agent
- **AND** 会话计数增加

#### Scenario: 会话超时清理
- **WHEN** 会话超过超时时间未活动
- **THEN** 会话被自动删除
- **AND** 过期会话计数增加
- **AND** 资源被释放

#### Scenario: 会话持久化
- **WHEN** 系统优雅退出
- **THEN** 会话数据保存到文件
- **AND** 系统重启后可恢复会话

### Requirement: 缓存管理系统

系统 SHALL 提供双层缓存管理功能：
- 支持内存缓存（默认最大 100MB）
- 支持磁盘缓存（默认最大 1GB）
- 支持 LRU 淘汰策略
- 支持缓存过期管理（默认 1 小时）
- 支持缓存键生成（基于 URL + 方法 + 关键请求头）
- 支持缓存统计信息查询
- 支持缓存预热

#### Scenario: 缓存命中
- **WHEN** 请求的资源在缓存中存在且未过期
- **THEN** 直接从缓存返回数据
- **AND** 缓存命中计数增加
- **AND** 跳过目标服务器请求

#### Scenario: 缓存未命中
- **WHEN** 请求的资源不在缓存中
- **THEN** 从目标服务器获取数据
- **AND** 数据被缓存
- **AND** 缓存未命中计数增加

#### Scenario: 缓存淘汰
- **WHEN** 缓存空间不足
- **THEN** 使用 LRU 策略淘汰最久未使用的缓存
- **AND** 淘汰计数增加
- **AND** 释放空间

#### Scenario: 缓存雪崩防护
- **WHEN** 大量缓存同时过期
- **THEN** 使用随机 TTL 防止雪崩
- **AND** 缓存过期时间分散

### Requirement: 黑名单拦截系统

系统 SHALL 提供多层次黑名单拦截功能：
- 支持 IP 黑名单（单 IP 和 IP 范围）
- 支持 URL 黑名单（精确匹配和正则匹配）
- 支持域名黑名单
- 支持白名单优先级机制
- 支持配置热重载
- 支持拦截统计信息查询
- 支持动态添加和移除黑名单项

#### Scenario: IP 黑名单拦截
- **WHEN** 客户端 IP 在黑名单中
- **THEN** 请求被拦截
- **AND** 返回 403 错误
- **AND** 记录拦截日志

#### Scenario: URL 正则匹配拦截
- **WHEN** 请求 URL 匹配黑名单正则表达式
- **THEN** 请求被拦截
- **AND** 返回 403 错误
- **AND** 记录匹配规则

#### Scenario: 白名单优先
- **WHEN** 客户端 IP 在白名单中
- **THEN** 即使在黑名单中也允许访问
- **AND** 记录白名单放行日志

#### Scenario: 配置热重载
- **WHEN** 黑名单配置文件被修改
- **THEN** 系统自动重新加载配置
- **AND** 新规则立即生效
- **AND** 不中断现有连接

### Requirement: 脚本注入系统

系统 SHALL 提供前端脚本注入功能：
- 支持脚本加载和管理
- 支持条件注入（基于 URL 模式和内容类型）
- 支持注入位置控制（head_start/head_end/body_start/body_end）
- 支持脚本优先级管理
- 支持脚本配置热重载
- 支持注入统计信息查询
- 支持动态添加和移除脚本

#### Scenario: 条件注入成功
- **WHEN** 响应内容类型为 text/html
- **AND** URL 匹配脚本配置的模式
- **THEN** 脚本被注入到指定位置
- **AND** 注入计数增加

#### Scenario: 脚本优先级排序
- **WHEN** 多个脚本匹配同一请求
- **THEN** 按优先级从高到低注入
- **AND** 保证脚本执行顺序

#### Scenario: 脚本热重载
- **WHEN** 脚本配置文件被修改
- **THEN** 系统自动重新加载配置
- **AND** 新脚本立即生效
- **AND** 清除脚本缓存

### Requirement: 配置文件扩展

系统 SHALL 扩展配置文件以支持 V2 功能：
- 扩展 databases/config.json 添加 V2 模块配置
- 创建 databases/blacklist.json 黑名单配置
- 创建 databases/scripts.json 脚本注入配置
- 支持配置项验证和默认值

#### Scenario: V2 配置加载
- **WHEN** 系统启动时加载配置
- **THEN** V2 模块配置被正确解析
- **AND** 缺失配置使用默认值
- **AND** 无效配置被检测并报错

### Requirement: 脚本库创建

系统 SHALL 提供示例脚本库：
- 创建 Scripts/dock.js 浮动面板脚本
- 创建 Scripts/progress.js 进度条脚本
- 创建 Scripts/target.js 目标定位脚本
- 创建 Scripts/zoom.js 图片缩放脚本

#### Scenario: 浮动面板脚本
- **WHEN** dock.js 被注入到页面
- **THEN** 页面右下角显示控制面板
- **AND** 面板可拖拽移动
- **AND** 面板可最小化

#### Scenario: 进度条脚本
- **WHEN** progress.js 被注入到页面
- **THEN** 页面顶部显示加载进度条
- **AND** 加载完成后自动隐藏

## MODIFIED Requirements

### Requirement: 代理转发引擎扩展

系统 SHALL 扩展代理转发引擎以集成 V2 模块：
- 集成连接池管理
- 集成线程池处理
- 集成会话管理
- 集成缓存管理
- 集成黑名单拦截
- 集成脚本注入

#### Scenario: V2 集成请求处理流程
- **WHEN** 客户端发送请求
- **THEN** 系统按以下顺序处理：
  1. 黑名单检查
  2. 会话管理
  3. 缓存检查
  4. 连接池获取连接
  5. 线程池处理 CPU 密集型任务
  6. 脚本注入
  7. 缓存结果

### Requirement: 主程序扩展

系统 SHALL 扩展主程序以支持 V2 模块：
- 初始化所有 V2 模块
- 启动定期清理任务
- 协调 V2 模块生命周期
- 扩展优雅退出流程

#### Scenario: V2 模块初始化
- **WHEN** 系统启动
- **THEN** 所有 V2 模块被正确初始化
- **AND** 定期清理任务启动
- **AND** 监控任务启动

## REMOVED Requirements

无移除的需求（V2 为增量更新）
