# 脚本注入器架构重构计划（修订版）

## 问题诊断

### 核心问题：脚本注入到 JS 文件中

**根本原因**：`inject_scripts` 方法没有验证输入内容是否真的是 HTML。

#### 问题场景示例
如果 JS 代码中包含类似 HTML 的字符串：
```javascript
var template = '<html><head></head><body></body></html>';
```
注入方法会错误地在 JS 代码中查找并注入脚本！

### 问题详情

#### 1. 内容验证缺失
- `inject_scripts` 方法接收参数名为 `html`，但未验证是否为有效 HTML
- 注入方法使用正则查找 `<head>`、`<body>` 等标签
- 任何包含这些字符串的内容都会被错误注入

#### 2. Content-Type 检查不严格
- `normal.py` 使用 `'text/html' in content_type` 检查
- 某些服务器可能返回错误的 Content-Type（如将 JS 文件标记为 `text/html`）

#### 3. 内容类型匹配问题
- `get_scripts_for_url` 使用 `content_type == allowed_type` 精确匹配
- 对于 `text/html; charset=utf-8`，匹配会失败
- 导致脚本无法正确注入到 HTML 页面

#### 4. 配置格式与成功案例不一致
- 当前使用对象格式，成功案例使用数组格式
- 注入位置命名差异：`head_start` vs `after_head_start`
- 缺少域名排除功能

## 重构方案

### 核心修复：添加 HTML 内容验证

在 `inject_scripts` 方法开始时，验证内容是否为有效 HTML：

```python

    # 1. 验证内容类型
    if not content_type or 'text/html' not in content_type.lower():
        return html
    
    # 2. 验证内容是否为有效 HTML（必须包含 <html 或 <!DOCTYPE）
    html_lower = html.lower().strip()
    if not (html_lower.startswith('<!doctype') or 
            html_lower.startswith('<html') or 
            '<html' in html_lower[:1000]):
        self._logger.debug(f"内容不是有效 HTML，跳过注入: {url}")
        return html
    
    # 3. 继续注入逻辑...
```

## 实施步骤

### 第一阶段：紧急修复（核心问题）

#### 1.1 修改 `modules/scripts.py` 的 `inject_scripts` 方法
添加 HTML 内容验证，防止注入到非 HTML 内容：

```python
async def inject_scripts(self, html: str, url: str, content_type: str = 'text/html') -> str:
    # 紧急修复：验证内容类型
    if not content_type or 'text/html' not in content_type.lower():
        self._logger.debug(f"内容类型不是 HTML，跳过注入: {content_type}")
        return html
    
    # 紧急修复：验证内容是否为有效 HTML
    html_check = html.lower().strip()[:2000]  # 只检查前 2000 字符
    is_valid_html = (
        html_check.startswith('<!doctype') or
        html_check.startswith('<html') or
        '<html' in html_check or
        ('<head' in html_check and '<body' in html_check)
    )
    
    if not is_valid_html:
        self._logger.debug(f"内容不是有效 HTML 结构，跳过注入: {url[:100]}")
        return html
    
    # 原有注入逻辑...
```

#### 1.2 修改 `modules/scripts.py` 的 `get_scripts_for_url` 方法
改进内容类型匹配：

```python
# 检查内容类型
content_types = conditions.get('content_types', [])
if content_types:
    content_match = False
    content_lower = content_type.lower()
    
    for allowed_type in content_types:
        allowed_lower = allowed_type.lower()
        
        if allowed_lower.endswith('*'):
            prefix = allowed_lower[:-1]
            if content_lower.startswith(prefix):
                content_match = True
                break
        elif allowed_lower in content_lower:  # 改为包含匹配
            content_match = True
            break
    
    if not content_match:
        continue
```

### 第二阶段：架构重构（参照成功案例）

2. **配置格式变更**：需要迁移现有配置

1. 使用多重验证条件（DOCTYPE、html 标签、head+body 组合）
2. 提供配置迁移指南
